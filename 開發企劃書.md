# 🚀 校園自治整合平台：Python 開發企劃書 (Roadmap)

## 📊 開發總覽
* **核心語言**：Python 3.12+
* **後端框架**：**FastAPI** (非同步高併發架構)
* **資料庫 ORM**：**SQLAlchemy 2.0** (搭配 Alembic 進行遷移)
* **專案管理**：**uv** (現代化高效能依賴管理與 Workspaces)
* **開發階段**：9 個階段，共 41 個功能模組

---

## 🛠 技術棧對照表（Node.js → Python 替代方案）

| 類別 | **Python 替代方案** | 說明 |
| :--- | :--- | :--- |
| **核心框架** | **FastAPI** | 同樣具備強類型檢查與依賴注入機制 |
| **包管理/Monorepo** | **uv Workspaces** | 2024 年後最快的 Python 整合工具 |
| **資料庫 ORM** | **SQLAlchemy + Alembic** | Python 業界標準，支援複雜關聯與異步 |
| **身份驗證** | **Authlib + PyJWT** | 完整的 OAuth2 與 JWT 支援 |
| **非同步任務佇列** | **Celery + Redis** | 處理 Email、統計任務的最強工具 |
| **即時通訊** | **FastAPI WebSockets** | 原生支援，無需額外包裝 |
| **資料匯出** | **Pandas + Openpyxl** | Python 的絕對領域，處理報表極快 |

## ⚡ 關鍵路徑（Critical Path）
> **P0 基礎建設** → **P1 認證系統** → **P2 組織權限** → **P3 共用服務** → **[ P4–P7 並行 ]** → **P8 整合收尾**

---

## 📁 PHASE 00 ｜ 基礎建設 (P0)
* **[M-00] uv Workspaces 專案骨架**：
    * 使用 `uv init --workspace` 初始化。
    * `apps/api`: FastAPI 主程式。
    * `apps/web`: Next.js 前端（保留強大的 React 生態）。
    * `libs/shared`: 共用 Python 類型定義 (Pydantic models) 與工具。
* **[M-01] SQLAlchemy 2.0 全域設計**：
    * PostgreSQL 建立，使用 **Alembic** 管理 Migration。
    * 設計 User, Org, Position, Permission 關聯模型。
* **[M-02] 容器化與 CI/CD**：
    * Dockerfile 優化（Multi-stage build）。
    * GitHub Actions 執行 Pytest 與 Ruff (Linting/Formatting)。

---

## 📁 PHASE 01 ｜ 身份驗證系統 (P1)
* **[M-03] OAuth2 / OpenID Connect**：
    * 整合 **Authlib** 處理 Google 登入。
    * 實作 FastAPI `OAuth2PasswordBearer`。
* **[M-04] JWT 安全機制**：
    * 實作雙 Token 機制（Access/Refresh）。
    * 使用 Redis 儲存黑名單（登出機制）。
* **[M-05] 使用者管理**：
    * FastAPI 依賴注入 (DI) 獲取當前使用者資訊。

---

## 📁 PHASE 02 ｜ 組織架構 ＆ 權限系統 (P2)
* **[M-06] 組織樹狀結構**：
    * 使用 SQLAlchemy 實作 `Adjacency List` 或 `Materialized Path`。
* **[M-07] 職位與身份**：
    * 一對多/多對多關聯，處理任期邏輯（Python `datetime` 處理）。
* **[M-08] FastAPI RBAC 權限引擎**：
    * **替代 NestJS Guard**：使用 FastAPI 的 `Depends` 機制實作權限檢查器。
    * 自定義 `PermissionChecker` 類別，實現 `@app.get(..., dependencies=[Depends(get_current_active_user)])`。

---

## 📁 PHASE 03 ｜ 共用服務層 (P3)
* **[M-10] Email 與 Celery 任務**：
    * 使用 **FastAPI-Mail** 整合非同步發送。
    * **Celery** 處理耗時任務，確保 API 不會阻塞。
* **[M-11] LINE SDK 整合**：
    * 使用官方 `line-bot-sdk-python`。
* **[M-13] WebSockets 即時推播**：
    * FastAPI 原生 `WebSocketRouter` 實作房間管理。

---

## 📁 PHASE 04 ｜ 公文與法規系統 (P4)
* **[M-17] 字號自動生成**：
    * 使用原子性操作確保序號不重複。
* **[M-18] 簽核狀態機**：
    * 使用 Python 的強大邏輯處理多層級審核。
* **[M-20] 法規系統 (Markdown 支援)**：
    * 整合 **Python-Markdown** 或 **Mistune** 進行解析。

---

## 📁 PHASE 05 ｜ 購票 / 校商訂購系統 (P5)
* **[M-23] 訂單系統與樂觀鎖**：
    * 利用 SQLAlchemy 的 `version_id_col` 實作樂觀鎖。
* **[M-26] 報表匯出 (Python 優勢)**：
    * **Pandas** 快速生成班級訂購統計，一鍵匯出 Excel/CSV。

---

## 📁 PHASE 06 ｜ 學餐訂購系統 (P6)
* **[M-28] 餐點排程**：
    * 利用 Python `calendar` 模組處理多日批量排程。
* **[M-30] 結單通知**：
    * Celery Beat 定時執行，每日自動結單。

---

## 📁 PHASE 07 ｜ 問卷與意見回報系統 (P7)
* **[M-32] 動態題型引擎**：
    * 使用 **Pydantic** 驗證動態 JSON 欄位（問卷內容）。
* **[M-35] 數據分析 (核心特色)**：
    * 可直接在後端整合 **NumPy** 進行問卷統計分析。
* **[M-37] 匿名隔離**：
    * 雙資料庫 Table 設計，確保填答人與答案無邏輯關聯。

---

## 📁 PHASE 08 ｜ 整合、優化與收尾 (P8)
* **[M-38] Swagger 自動化文件**：
    * FastAPI 自動生成 `/docs`，無需手動配置 Swagger。
* **[M-39] 效能調優**：
    * 使用 **Gunicorn + Uvicorn workers** 部署。
    * **Redis** 快取熱點資料。
